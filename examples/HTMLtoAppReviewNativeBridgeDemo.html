HTML to App Native Integration Review Demo

This file is made for HTML to App. It is designed to be exported into a macOS app bundle, then used to test native bridge features that a browser and web clip cannot provide.

This file is not meant to run in a browser

If this page is opened in Safari, Chrome, or another browser, it can only render static HTML. The native macOS bridge is not available.

To test this file, create an app bundle with HTML to App, enable Open With support, export the app, approve it through the standard macOS flow, and launch the exported app.

Expected result in a browser: bridge features are disabled. This is intentional. A browser cannot create a native menu bar icon, control a Dock badge, register Finder Open With file types, or receive Finder-opened files in JavaScript.

Reviewer workflow

This page is now running inside the exported HTML to App app bundle. Use the steps below to test features that are not possible in a regular browser or web clip.

  1. Press Request notification permission, then use the Dock badge and native notification buttons once they unlock.
  2. Press Create native menu bar item. A macOS menu bar icon and native menu should appear, and the menu bar badge/window buttons should unlock.
  3. Press Destroy menu bar item when you want to remove the status item and start the menu bar test again.
  4. In Finder, right-click this same HTML file, choose Open With, then select the exported app bundle.
  5. Return to the app. The file opened from Finder should appear in the Finder Open With passthrough section.
  6. Select the received file and press Read selected file through bridge. The app reads the Finder-opened file through a scoped native bridge.

Notification permission

Native notifications and Dock badge

macOS only shows the Dock badge of an exported app once it has notification permission, so notifications and the Dock badge are controlled together. Grant native notification permission first; the notification and Dock badge buttons unlock once permission is granted. A browser cannot directly control the Dock badge of an exported standalone app.

Notification permission is required before notification and Dock badge actions are enabled.

Native menu bar icon and menu

This is a native macOS status item created from JavaScript. A browser tab or web clip cannot add a native menu bar item.

Test this by opening this same HTML file from Finder with the exported app bundle. The file should be delivered to JavaScript as an opened item.
